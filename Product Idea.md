Product Idea
What If...? Heritage on K2
Think V2 Interactive alternate history simulator styled after Marvel’s What If. Users input a
“what-if” scenario; the agent generates branching timelines with detailed chain-of-thought
reasoning, visualizing alternate realities through interactive timelines, maps, and branching
consequence trees.


🔑 API Access

API Key is stored in `frontend/.env.local` (not committed to version control).
See `frontend/.env.example` for required environment variables.

API endpoint: `https://api.k2think.ai/v1/chat/completions`
Model: `MBZUAI-IFM/K2-Think-v2`

## 📋 План реализации

### Концепция
Пользователь вводит альтернативный исторический сценарий ("Что если бы Наполеон выиграл при Ватерлоо?").
K2 Think V2 генерирует дерево последствий — каждое событие порождает ветви с цепочкой
причинно-следственных связей. Пользователь может кликать на ветви и углубляться в них,
генерируя новые подветви. Визуально — интерактивное дерево таймлайнов в стиле Marvel "What If...?"
(тёмная космическая тема, порталы, параллельные реальности).

---

### Архитектура

```
[Пользователь] → [Next.js Frontend] → [Next.js API Routes] → [K2 Think V2 API]
                      │                        │
                  React Flow              Промпт-инжиниринг
                  (дерево ветвей)         (структурированный JSON)
```

**Почему API Routes вместо отдельного бэкенда?** Next.js 16 уже имеет серверные API Routes —
не нужен отдельный Express сервер. API ключ хранится в .env на сервере, клиент его не видит.
Отдельный `backend/` папка не нужна.

### Этапы разработки

#### Этап 1 — Фундамент (Backend + Промпты)
1. Настроить `.env.local` с API ключом K2
2. Создать API Route `app/api/generate/route.ts` — проксирует запрос к K2 Think V2
3. Написать системный промпт, который заставляет K2 возвращать структурированный JSON
   с деревом альтернативных последствий
4. Поддержка стриминга (SSE) для плавного отображения генерации
5. Создать API Route `app/api/expand/route.ts` — для углубления в конкретную ветку

#### Этап 2 — UI: Главная страница
6. Тёмная космическая тема (градиенты, звёзды, свечения) в стиле Marvel What If
7. Большой input для ввода сценария ("Что если бы...")
8. Примеры сценариев (кнопки-подсказки) для быстрого старта
9. Кнопка "Создать реальность" с анимацией портала

#### Этап 3 — Визуализация дерева
10. Интеграция React Flow для отображения дерева ветвей
11. Кастомные ноды — каждая ветвь как карточка с годом, названием, описанием
12. Цветовая кодировка по уровню влияния (critical/high/medium/low)
13. Анимация появления новых веток (Framer Motion)

#### Этап 4 — Интерактивность
14. Клик на ветку → раскрытие деталей в боковой панели
15. Кнопка "Углубиться" → запрос к K2 → генерация подветвей в реальном времени
16. Зум, перемещение по дереву, авто-раскладка

#### Этап 5 — Полировка
17. Loading состояния и скелетоны при генерации
18. Адаптив под мобильные устройства
19. Историческая справка — AI объясняет реальный исход для сравнения

### Ключевые решения

**Промпт-стратегия**: Системный промпт просит K2 Think V2 думать пошагово (chain-of-thought),
а затем выдавать результат в строгом JSON формате. Reasoning модели K2 идеально подходят для
построения причинно-следственных цепочек.

**Expand (углубление)**: Когда пользователь кликает "углубиться" на ветке, мы отправляем
контекст всей цепочки от корня до текущей ветки, и K2 генерирует новые подветви.
Это даёт бесконечную глубину исследования.

**Стриминг**: K2 API поддерживает stream=true. Мы используем SSE чтобы показывать
генерацию текста в реальном времени — пользователь видит как "реальность создаётся".
